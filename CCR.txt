CHANGE CONTROL REQUEST (CCR)
e-Government Authority (eGA) Compliance Document

1. Executive Summary of the Change
---------------------------------------------------------
Title: Deployment of TGDC Website version 1.0.
Reason for Change: Transition from a verified development/staging environment to a live public-facing government portal.
Urgency: Planned

2. Technical Scope (The "What")
---------------------------------------------------------
• Source Code: Branch `main` (Target commit hash: bfe64c25857e122bfb3363a63c192c1cd8750950 via GitHub repository).
• Database: Prisma schema migrations to initialize production database tables (MySQL), creation of default users, and initial content seeds.
• Environment Specs: Node.js (Next.js v15.5.x, React v19.x), Tailwind CSS v4.x, and Prisma ORM. Deployed on a Linux server environment (Management IP: 100.127.7.37, requiring ports 80/443 open to the public internet).
• Environment Variables: Environment variables (.env) have been pre-configured on the production server to point to production-grade secrets, ensuring no development keys are used in the live environment.

3. Impact Analysis
---------------------------------------------------------
• Services Affected: The old website (https://www.tgdc.go.tz/) will be replaced; routing will transition to the new application.
• Downtime Estimate: Approximately 15 to 30 minutes of scheduled maintenance window during final DNS propagation and build execution.
• Security Posture: This deployment strictly follows the successful security audit recently passed. 

4. Implementation Plan (The "How")
---------------------------------------------------------
[ ] 1. Backup existing database and files from the legacy TGDC website.
[ ] 2. Establish SSH connection to the new production server (IP: 100.127.7.37).
[ ] 3. Trigger CI/CD pipeline from the GitHub repository (`main` branch) to handle the CPU-intensive Next.js framework build process securely off the live server.
[ ] 4. Deploy the pre-built, optimized artifacts generated by the CI/CD pipeline to the production instance.
[ ] 5. Run Prisma migrations against the production database (`npx prisma migrate deploy`).
[ ] 6. Start the web application service orchestrator (e.g., PM2 or Docker container).
[ ] 7. Update DNS/SSL certificates to direct traffic from www.tgdc.go.tz to the new server IP and ensure HTTPS traffic is encrypted correctly.
[ ] 8. Execute formal Verification Test (Health check, CMS login test, stakeholder authentication test).

5. Rollback Plan (The "Emergency Exit")
---------------------------------------------------------
• Trigger: If the site returns a 500 error or becomes unresponsive for more than 5 minutes post-deployment.
• Action: Revert to the previous stable version of the code, restore the database from the backup taken in Step 1, and map the DNS temporarily back to the old server infrastructure if necessary.
