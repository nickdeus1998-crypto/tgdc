2. SECURITY VULNERABILITIES & MISCONFIGURATIONS
-----------------------------------------------

2.1 [CRITICAL] UFW Firewall is INACTIVE
    Risk: The server's built-in Uncomplicated Firewall (UFW) is disabled.
    Impact: All network ports bound to 0.0.0.0 are exposed to the internet,
            bypassing basic access controls.
    Recommendation:
      - Enable UFW immediately: `sudo ufw enable`
      - Allow OpenSSH (Port 22/TCP)
      - Allow Nginx (Port 80/TCP, 443/TCP)
      - Deny all other incoming traffic by default.

2.2 [HIGH] SSH Password Authentication is ENABLED
    Risk: `sshd_config` permits PasswordAuthentication.
    Impact: The server is vulnerable to SSH brute-force attacks (dictionary attacks).
    Recommendation:
      - Set `PasswordAuthentication no` in `/etc/ssh/sshd_config`.
      - Enforce Key-Based Authentication (RSA/Ed25519) for `tgdc-admin`.
      - Set `PermitRootLogin prohibit-password` or `no`.

2.3 [MEDIUM] Suspicious Internal Port Bindings
    Observation:
      - Port 3000 (Node.js/PM2) is listening on `0.0.0.0`.
      - Port 44221 (Node/Prisma studio?) is listening on `127.0.0.1`.
    Risk: If Node.js is meant to be proxied behind Nginx, it should only bind
          to `127.0.0.1:3000`, not `0.0.0.0:3000`. Exposing 3000 globally allows
          attackers to bypass Nginx security headers, rate limiters, or WAFs.
    Recommendation:
      - Update the start script to bind to localhost: `HOST=127.0.0.1 pm2 start npm --name "tgdc" -- run start`

2.4 [MEDIUM] Node.js Version (v18.20.8) is Nearing EOL
    Risk: Node 18 reaches End-of-Life (EOL) in April 2025.
    Impact: No further security patches will be provided. We already encountered
            compatibility issues with `isomorphic-dompurify`.
    Recommendation:
      - Upgrade the server to Node.js v20 LTS or v22 LTS.

3. PROCESS MANAGEMENT & STABILITY
---------------------------------------------

3.1 [MEDIUM] PM2 App Instability (80 Restarts)
    Observation: The `tgdc` PM2 process shows 80 process restarts.
    Impact: High restart counts indicate the application is frequently crashing,
            likely due to Out-Of-Memory (OOM) errors, unhandled promise rejections,
            or SQLite locking conflicts.
    Recommendation:
      - Inspect PM2 logs: `pm2 logs tgdc --lines 100` to identify the crash reason.
      - SQLite database locking (`P2024`) is a common cause in Next.js when multiple
        requests attempt simultaneous writes.
      - Consider running PM2 in cluster mode (`-i max`) ONLY if migrated to PostgreSQL/MySQL.
        SQLite cannot handle multi-process concurrency safely.
